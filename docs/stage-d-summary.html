Stage D — Daily Streaks 🔥

A gentle, everyday reason to come back: students build a streak for each day they finish work, and the whole school can see their flame grow.

Shipped · Live on main & production

What students get

🔥

A daily streak

Every day a student completes at least one assignment, their streak grows by one. We track their current streak and their best-ever record.

⭐

The flame, front and centre

A small flame and day-count sit right next to the XP bar on every student’s dashboard — primary and Forms — loading instantly with the page.

❄️

Streak freezes

Miss a day and a freeze is used automatically to save the streak — “Your streak freeze saved you!” Students earn one each time they level up, holding up to two.

🏅

Milestone rewards

Reaching 3, 7, 14 and 30 days triggers a one-time celebration and a bonus of XP — always within the daily XP cap, so nothing can be farmed.

🌍

Their real day

Days are counted in Mozambique time (CAT, UTC+2), so “today” lines up with the children’s actual day — no confusing midnight cut-offs.

💛

Kind, never shaming

If a streak is lost with no freeze left, the message encourages: “Start a new streak today — your record is 12 days!” We keep the record to cheer them on.

Under the hood

  • •Self-contained. Streaks are a brand-new module — auto-marking, XP, and the Treasure Island were left untouched.
  • •No extra loading. The flame rides along with the dashboard’s existing data request — zero additional network calls.
  • •Never in the way. Streak updates are best-effort, so a hiccup can never block a submission or break the dashboard.
  • •No background jobs. A missed day is worked out the moment a student is next active or opens their dashboard.

Also fixed on the way

A production issue was caught and fixed during this work.

Verifying the deploy surfaced a live error on the student dashboard: a database table the new feature needed hadn’t been created during deployment, which made the dashboard’s stats fail to load.

It’s now fixed two ways — the dashboard shrugs off any gamification hiccup instead of failing, and the app now guarantees these tables exist on start-up. This also quietly repaired XP and rewards tables that an earlier deploy had likely missed, so this class of problem can’t recur.

For the developers — how to test

Run npm run dev, then bash script/test-streaks.sh. It uses dev-only endpoints to simulate day changes, so you can watch a streak build, hit a milestone, lose a day, and be saved by a freeze — all without waiting real days. These test endpoints are automatically disabled in production.
